GAMELANG : an overview

Hi! This is meant as an introduction / explanation for my proof-of-concept language: GAMELANG
Let's start with the basics:

Whitespace

Spaces are the foundation of parsing for this language, an item (or "token") to be distinguishable from its neighbours must be well spaced out on both sides.
That said indentation is instead irrelevant, just like the amount of spaces you leave around your tokens.

Types

GAMELANG is a dynamicly typed, interpreted and synchronous language with only four proper primitive types:
  • NUMBER - GAMELANG does not distinguish between numbers and booleans, namely:
    0 ←→ false , 1 ←→ true.
  • STRING - Currently, support for escaping characters is limited to "&new", which replaces the much more common "\n".
    "Hello, World!&newWhat's up?" # >> Hello, World!
                                  #    What's up?
  • LIST - A collection of indexable items, of any type (even other lists).
  • OBJECT - A collection of "property : value" pairs that have to be indexed by name and are not positional.
    Object properties are more precisely called "methods" if their access operation corresponds to a "Function call", but we'll get to that later.
What you can also notice is the absence of a NULL type of sorts. In fact, such values aren't allowed in the language and while it's possible to create an empty string ("") similar structures for other types, such as an empty list ([]) or object ({}), are impossible to obtain.

Expressions

Any line of code written in GAMELANG is called an Expression, of which there are 3 valid types:
  • Procedures: a procedure, like any other expression, follows a specific pattern of arguments and syntactic rules. Most notably, it always begins with a keyword.
    This is a list of all currently available keywords:
    • print
    • make
    • free
    • when
    • else
    • options
    • loop
    • next
    • exit
    • def
    • use
    • with
    • replace
    • flag
    Definition: <Print> := print <StackExpression>
    Usage: Prints its value to the console.

    This is the first procedure we tackle, so let's look at a couple examples:

    print 2   # this value is a NUMBER
    print "2" # this value is a STRING
    print 1 2 "hi" 3 # these values are MANY >> [ 1, 2, hi, 3 ]


    As you can see from example #3, multiple values share the same space with no separators. Moreover, we aren't performing any operations on them, which results in having to deal with multiple (as the comment suggest: MANY) values. When this happens what you obtain is a LIST.

    To understand why this happens, we need to talk about the only (not entirely true) argument this procedure takes in: a "StackExpression".
    Stack expressions are the way values are created, changed and manipulated in this language. Anytime a value is written, transferred, changed, calculated or even taken as an input from the user, it is bound to pass through a stack expression.
    So how does it work?
    Every time a stack expression argument is executed, 3 things happen in order:

    1) An empty stack is created. This

    2) evaluate all of the stack elements you wrote, in order: this means that operators such as "+" must go AFTER both its inputs were put on the stack. this is called "polish notation", and ensures linear priority in evaluating mathematical and logical operations, making them always inequivocably objective in terms of parsing. No liberties / conventions have been taken by me in developing a parser for <StackExpression>s , everything is linear, clear, expected. #division example

    3) decide how to "cast" the output based on its length: no <StackExpression> is ever allowed to end its evaluation empty in this language, but besides that if more than one element is left on the stack after all operations have performed, the return value of that <StackExpression> will be a list of all remaining elements A few examples: print 1 2 + >> 3 print 1 "2" + >> "12" print 1 2 3 >> [ 1, 2, 3 ]
